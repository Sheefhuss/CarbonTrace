## Overview

CarbonTrace is a full-stack web application that helps individuals track, understand, and reduce their personal carbon footprint. Users log daily activities across three categories — Transport, Food, and Shopping — and receive real-time CO₂ equivalent estimates based on verified emission factors. The app compares individual footprints against the Paris Agreement 2030 target (2.5 tonnes CO₂e/year) and the India national average (150 kg/month).

The platform includes a gamification layer with points, badges, weekly challenges, and a community leaderboard, along with AI-powered personalised tips generated by Google Gemini and an interactive carbon assistant chatbot powered by Groq AI (LLaMA 3.3 70B).

## Features

###  Activity Logging
- Log transport, food, and shopping activities with real-time CO₂ preview
- Colour-coded impact levels — zero → low → medium → high
- Real-world equivalents shown (km driven, trees needed, kWh used)
- Low-carbon bonus points for activities under 2 kg CO₂e
- Dashboard updates instantly after saving — no page refresh needed

###  Personal Dashboard
- Monthly CO₂ total compared against India and world averages
- Paris 2030 target benchmark on every dashboard
- Emissions breakdown by category with donut chart
- Monthly trend chart over last 6 months
- Location detection for localised comparisons via OpenStreetMap

###  Gamification
- Points system with daily login rewards and activity bonuses
- 12 achievement badges with milestone tracking
- 6 weekly challenges that reset every Monday
- Community leaderboard scored on activity count, streak, emission reduction, and badges
- Friends system with unique 8-character friend codes — both users must connect mutually

###  AI Features
- **AI Tips** — Google Gemini 2.0 Flash generates 3 personalised tips daily based on actual emission data, cached in localStorage to avoid quota exhaustion
- **CarbonBot** — Floating interactive chatbot powered by Groq (LLaMA 3.3 70B), knows all CarbonTrace emission factors, point values, and challenge details

###  User System
- Two-step registration with 15 cartoon avatar options
- Login with email or @username — email never exposed publicly
- Streak tracking with longest streak record
- Weekly points reset every Monday for competitive leaderboard

###  Footprint Quiz
- 3-step onboarding quiz covering home, transport, and diet
- Estimates annual baseline footprint used as context for AI tips

## Getting Started

### Prerequisites
- Node.js 18+
- PostgreSQL 14+
- npm

### 1. Clone the Repository

```bash
git clone https://github.com/yourusername/carbontrace.git
cd carbontrace
```

### 2. Set Up the Database

```bash
psql -U postgres
CREATE DATABASE carbon_footprint;
\q
```

### 3. Set Up the Backend

```bash
cd backend
npm install
```

Create `backend/.env` (see Environment Variables section below), then:

```bash
node migrate.js
node migrate_v2.js
npm run dev
```

Backend runs on **http://localhost:5000**

### 4. Set Up the Frontend

```bash
cd frontend
npm install
```

Create `frontend/.env` (see Environment Variables section below), then:

```bash
npm run dev
```

Frontend runs on **http://localhost:3000**

### 5. Getting API Keys

**Gemini API (AI Tips — free):**
1. Go to [aistudio.google.com](https://aistudio.google.com)
2. Click **Get API Key** → Create API Key
3. Key starts with `AIza...`

**Groq API (CarbonBot — free):**
1. Go to [console.groq.com](https://console.groq.com)
2. Sign up → API Keys → Create API Key
3. Key starts with `gsk_...`

## Deployment

### Recommended Free Stack

```
User → Vercel (React frontend)
         ↓
   Render (Node/Express API)
         ↓
   Neon (PostgreSQL database)
```

### Step 1 — Database (Neon)
1. Create free account at [neon.tech](https://neon.tech)
2. Create project → copy connection string
3. Export local DB: `pg_dump -U postgres carbon_footprint > backup.sql`
4. Import to Neon via their SQL editor

### Step 2 — Backend (Render)
1. Push code to GitHub
2. Go to [render.com](https://render.com) → New Web Service
3. Settings: Root `backend` | Build `npm install` | Start `npm start`
4. Add all backend env variables, set `DB_SSL=true`, use Neon DB credentials
5. Set `FRONTEND_URL=https://your-app.vercel.app`

### Step 3 — Frontend (Vercel)
1. Go to [vercel.com](https://vercel.com) → New Project
2. Settings: Root `frontend` | Build `npm run build` | Output `dist`
3. Add `VITE_GEMINI_API_KEY`, `VITE_GROQ_API_KEY`
4. Add `VITE_API_URL=https://your-backend.onrender.com/api`

### Notes
- Render free tier sleeps after 15 min inactivity — first wake takes ~30 sec
- Use [UptimeRobot](https://uptimerobot.com) (free) to ping backend every 10 min to prevent sleep
- Neon free tier: 0.5 GB storage — sufficient for this project
